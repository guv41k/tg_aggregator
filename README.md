# tg_aggregator

Система автоматического сбора корпоративных переписок Telegram с выгрузкой по команде администратора.

## Как это работает

- **Бот-сборщик** добавляется в корпоративные чаты и сохраняет все сообщения (текст, медиа, реакции) в PostgreSQL; медиафайлы скачиваются локально в папку `media/`.
- **Бот-администратор** предоставляет кнопочный интерфейс администраторам для выгрузки истории переписки за указанный период в ZIP-архив (текст + вложения).

## Переменные окружения

Создай файл `.env` в корне проекта по образцу `.env.example`:

| Переменная | Описание | Пример |
|---|---|---|
| `BOT_TOKEN` | Токен бота-администратора (получить у @BotFather) | `7123456789:AAF...` |
| `COLLECTOR_TOKEN` | Токен бота-сборщика (получить у @BotFather) | `7987654321:AAG...` |
| `DATABASE_URL` | Строка подключения к PostgreSQL | `postgresql+psycopg2://user:pass@localhost:5432/tg_aggregator` |
| `ADMIN_IDS` | Telegram user_id администраторов через запятую | `123456789,987654321` |

## Установка и запуск

### 1. Клонировать репозиторий

```bash
git clone <url>
cd tg_aggregator
```

### 2. Создать виртуальное окружение и установить зависимости

```bash
python -m venv .venv
# Windows
.venv\Scripts\activate
# Linux / macOS
source .venv/bin/activate

pip install -r requirements.txt
```

### 3. Настроить окружение

```bash
cp .env.example .env
# Заполнить .env своими значениями
```

### 4. Создать базу данных в PostgreSQL

```sql
CREATE DATABASE tg_aggregator;
```

### 5. Запустить

```bash
python main.py
```

При первом запуске таблицы создаются автоматически.

## Интерфейс бота-администратора

Управление полностью кнопочное. Единственная команда — `/start`.

| Кнопка | Действие |
|---|---|
| 📋 Список чатов | Показывает все чаты, сохранённые в базе |
| 📤 Выгрузка | Запускает диалог экспорта: выбор чата → дата начала → дата конца |
| ← Назад | Возврат на главный экран |
| ❌ Отмена | Прервать диалог выгрузки в любой момент |

После ввода дат бот формирует и отправляет ZIP-архив с файлом `export.txt` и всеми медиавложениями за выбранный период (лимит — 50 МБ).

## Архитектура

```
tg_aggregator/
├── main.py          # точка входа, запускает оба бота
├── config.py        # загрузка переменных окружения
├── db/              # модели, сессия, CRUD-функции
├── collector/       # бот-сборщик сообщений
├── admin_bot/       # бот-администратор
└── utils/           # форматирование выгрузки
```
